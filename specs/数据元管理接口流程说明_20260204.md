## 一、业务概述

### 1.1 功能描述

数据元管理模块用于管理系统中的数据元（Data Element），支持以下核心功能：

- **数据元创建**：支持新增单个数据元及批量导入
- **数据元查询**：支持按目录、关键字、状态、类型等多维度分页查询
- **数据元编辑**：支持修改数据元信息，含版本控制
- **数据元删除**：支持单个或批量删除
- **启用/停用**：支持批量启用/停用操作
- **目录移动**：支持批量移动数据元到指定目录
- **关联管理**：支持关联码表、编码规则、标准文件
- **导入导出**：支持Excel导入导出

### 1.2 基础路径

```
基础路径：/api/standardization/v1/dataelement
```

## 二、数据库表结构

### 2.1 数据元基本信息表（t_data_element_info）

```sql
CREATE TABLE `t_data_element_info` (
  `f_id` bigint(20) NOT NULL AUTO_INCREMENT COMMENT '主键',
  `f_de_id` bigint(20) NOT NULL COMMENT '关联标识',
  `f_name_en` varchar(128) NOT NULL COMMENT '英文名称',
  `f_name_cn` varchar(128) NOT NULL COMMENT '中文名称',
  `f_synonym` varchar(300) DEFAULT NULL COMMENT '同义词',
  `f_std_type` INT(2) NOT NULL COMMENT '标准分类',
  `f_data_type` INT(2) NOT NULL COMMENT '数据类型',
  `f_data_length` INT(4) DEFAULT NULL COMMENT '数据长度',
  `f_data_precision` INT(4) DEFAULT NULL COMMENT '数据精度',
  `f_dict_code` bigint(20) DEFAULT NULL COMMENT '关联码表编码',
  `f_rule_id` bigint(20) DEFAULT NULL COMMENT '关联编码规则ID',
  `f_relation_type` VARCHAR(50) DEFAULT NULL COMMENT '关联类型：no/codeTable/codeRule',
  `f_catalog_id` bigint(20) NOT NULL COMMENT '目录ID',
  `f_label_id` bigint(20) DEFAULT NULL COMMENT '数据分级标签ID',
  `f_description` varchar(300) DEFAULT NULL COMMENT '说明',
  `f_version` INT(4) NOT NULL DEFAULT 1 COMMENT '版本号',
  `f_state` INT(2) NOT NULL DEFAULT 1 COMMENT '状态：1-启用，0-停用',
  `f_authority_id` varchar(100) DEFAULT NULL COMMENT '权限域',
  `f_department_ids` varchar(350) DEFAULT NULL COMMENT '部门ID',
  `f_third_dept_id` varchar(36) DEFAULT NULL COMMENT '第三方部门ID',
  `f_create_time` datetime DEFAULT NULL COMMENT '创建时间',
  `f_create_user` varchar(128) DEFAULT NULL COMMENT '创建用户',
  `f_update_time` datetime DEFAULT NULL COMMENT '修改时间',
  `f_update_user` varchar(128) DEFAULT NULL COMMENT '修改用户',
  `f_deleted` bigint(20) NOT NULL DEFAULT 0 COMMENT '逻辑删除标记',
  PRIMARY KEY (`f_id`),
  KEY `idx_de_id` (`f_de_id`),
  KEY `idx_dict_code` (`f_dict_code`),
  KEY `idx_rule_id` (`f_rule_id`)
) COMMENT='数据元基本信息表';
```

### 2.2 数据元-文件关系表（t_relation_de_file）

```sql
CREATE TABLE `t_relation_de_file` (
  `f_id` bigint(20) NOT NULL COMMENT '主键',
```

## 五、接口详细说明

### 5.1 创建数据元

**接口路径**：`POST /api/standardization/v1/dataelement`

#### 请求参数

```json
{
  "nameEn": "userName",
  "nameCn": "用户名",
  "synonyms": "账户,账号",
  "stdType": 0,
  "dataType": 1,
  "dataLength": 50,
  "dataPrecision": 0,
  "dictId": 123,
  "ruleId": 456,
  "relationType": "codeTable",
  "catalogId": 33,
  "stdFiles": [1, 2, 3],
  "description": "用户名称"
}
```

#### 业务逻辑

```
1. 参数校验
2. 业务校验（目录存在性、中英文名称唯一性）
3. 关联校验（码表/编码规则是否存在）
4. 数据处理（生成ID、设置创建信息）
5. 保存数据库
6. 保存关联文件关系
7. 发送MQ消息（topic: MQ_MESSAGE_SAILOR）
```

#### 异常提示

| 序号 | 错误码 | 字段 | 提示信息 | 触发场景 |
|------|--------|------|----------|----------|
| 1 | InvalidParameter | catalog_id | 数据元对应的目录不存在 | 目录不存在 |
| 2 | InvalidParameter | catalog_id | 数据元对应的目录类型不正确 | 目录类型非数据元 |
| 3 | InvalidParameter | relationType | 数据元关联类型不能为空 | relationType为空 |
| 4 | InvalidParameter | rule_id | 数据元关联编码规则不能为空 | relationType=codeRule且ruleId为空 |
| 5 | InvalidParameter | dict_id | 数据元关联码表不能为空 | relationType=codeTable且dictId为空 |
| 6 | InvalidParameter | rule_id | 编码规则不存在或已删除 | ruleId指向的规则不存在 |
| 7 | InvalidParameter | dict_id | 码表数据不存在或已删除 | dictId指向的码表不存在 |
| 8 | InvalidParameter | name_cn | 中文名称+标准分类不能全部重复 | 名称重复 |
| 9 | InvalidParameter | std_type | [标准分类]:输入错误 | stdType枚举无效 |
| 10 | InvalidParameter | data_type | [数据类型]:输入错误 | dataType枚举无效 |
| 11 | InvalidParameter | name_en | [英文名称]:空 | nameEn为空 |
| 12 | InvalidParameter | name_en | [英文名称]:字符不符合要求或长度超过128 | nameEn格式错误 |
| 13 | InvalidParameter | name_cn | [中文名称]:空 | nameCn为空 |
| 14 | InvalidParameter | name_cn | [中文名称]:长度超过128 | nameCn过长 |
| 15 | InvalidParameter | data_length | [数据长度]:输入错误 | 数字型长度>38或<1 |
| 16 | InvalidParameter | data_length | [数据长度]:输入错误 | 字符型长度>65535 |
| 17 | InvalidParameter | std_files | 一个或多个文件id不存在 | stdFiles包含不存在的文件ID |

### 5.2 批量导入数据元

**接口路径**：`POST /api/standardization/v1/dataelement/import`

#### 请求参数

- multipart/form-data：file（xlsx/xls文件），catalog_id

#### 特殊说明

- 单次导入限制：**最多5000条**
- 支持文件格式：.xlsx, .xls
- 文件大小限制：10M

#### 业务逻辑

```
1. 校验目录存在性
2. 校验文件类型和大小
3. 解析Excel文件
4. 逐行校验数据
5. 批量保存校验通过的数据
6. 返回导入结果（成功/失败列表）
7. 发送MQ消息
```

### 5.6 编辑数据元

**接口路径**：`PUT /api/standardization/v1/dataelement/{id}`

#### 请求参数

同创建数据元

#### 业务逻辑

```
1. 校验数据元存在性
2. 校验目录存在性
3. 校验关联关系
4. 判断版本是否变更
   - 关键属性变更（名称/关联类型/关联对象）：版本号+1
   - 非关键属性变更：仅更新当前版本
5. 更新数据
6. 更新关联文件关系
7. 发送MQ消息
```

## 六、值域计算规则

### 6.1 码表关联

当数据元关联码表时，值域格式为：`[码值1,码值2,...]`

```
示例：[M,F]
```

### 6.2 数据类型计算

当无码表关联时，根据数据类型计算值域：

| 数据类型 | 计算公式 |
|----------|----------|
| Number | (-(10^(长度-精度)-1), 10^(长度-精度)-1) |
| Decimal | 同Number |

## 七、版本控制规则

### 7.1 版本递增场景

修改以下属性时版本号+1：
- 关联类型（relationType）
- 关联对象（dictCode/ruleId）
- 名称（nameCn/nameEn）

### 7.2 版本不递增场景

修改以下属性时版本号不变：
- 目录ID（catalogId）
- 数据类型（dataType）
- 数据长度/精度
- 说明（description）

### 10.1 名称校验
